# Architecture

SignalDeck is intentionally thin: it does not replace OpenWebRX DSP, WebSocket, WebAudio, waterfall rendering, or decoder engines. It hosts OpenWebRX in an Android `WebView` and adds a mobile control layer.

## Main Components

- `MainActivity.java` - WebView host, native control deck, receiver list, OpenWebRX skin injection, debug logging.
- `TuningKnobView.java` - custom rotary tuning encoder.
- `ReceiverCatalog.java` - loader/parser for public receiver entries.
- `ReceiverInfo.java` - receiver model.
- `PlaybackKeepAliveService.java` - foreground service used for best-effort background playback.

## Control Strategy

SignalDeck combines two approaches:

- Native Android views for frequent controls such as tuning, SDR list, zoom, SQ and NR.
- Injected JavaScript/CSS for trimming desktop OpenWebRX UI and adapting receiver panels.

Native controls should be preferred when interaction must stay reliable across phones. Injected WebView styling should stay conservative because public OpenWebRX receivers use different markup.

## Receiver Switching

Receiver switching recreates WebView state where needed because some public receivers can leave blank or stale layers behind after navigation.

## Decoder Output

Decoder output is still generated by OpenWebRX. SignalDeck may style tables, but it should not hide, replace, or aggressively reparent decoder panels unless the server markup has been verified.
